## 3. 已冻结的关键结论

以下结论建议视为本轮开发冻结项：

### 3.1 产品形态

1. 首版是 `安卓APP`
2. 不是小程序
3. 不做 iOS 首发

### 3.2 主流程

1. 主入口是 `任务/桥梁`
2. 主链路是 `桥梁/项目 -> 构件 -> 采集 -> AI识别 -> 人工校核 -> 报告`
3. 不再沿用“首页直接拍照是主流程”的旧逻辑

### 3.3 MVP边界

1. 做现场检测闭环
2. 做离线存储与同步
3. 做报告生成与PDF导出
4. 不做无人机
5. 不做大屏
6. 不做复杂权限平台
7. 不做第三方系统集成

### 3.4 技术路线

1. 安卓原生 Kotlin
2. Jetpack Compose
3. Room + 文件系统 + DataStore
4. WorkManager 负责同步
5. CameraX 负责拍照录像
6. AI 引擎抽象，首版先 mock

### 3.5 数据结构

1. 检测记录与媒体拆表
2. `clientRecordId` 和 `clientMediaId` 必须存在
3. `syncStatus` 必须落库
4. AI任务单独建表
5. 报告内容使用结构化 JSON

## 7. 推荐第一个编码里程碑

如果现在正式开始写代码，建议第一个里程碑目标定为：

`安卓端在无真实后端、无真实AI的情况下，完整跑通桥梁 -> 项目 -> 构件 -> 采集 -> 本地保存 -> 构件结果查看。`

这是最稳的起点，因为它直接验证：

1. 页面结构是否正确
2. Room设计是否合理
3. 采集链路是否可靠
4. 构件归属是否清晰

---

## 8. 当前不建议再反复讨论的事项

为了避免继续消耗在前置阶段，以下话题建议暂不继续来回讨论：

1. 要不要做小程序
2. 要不要首发 iOS
3. 要不要先上无人机
4. 要不要直接做 Web 平台
5. 要不要一开始就做大屏和分析系统
6. 要不要把所有权限体系一次性做完

这些都不是当前MVP编码启动的必要条件。
